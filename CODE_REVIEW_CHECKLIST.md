# CODE_REVIEW_CHECKLIST.md

## 1) 需求与范围
- [ ] 需求目标清晰
- [ ] 仅改动必要模块（避免蔓延）
- [ ] 高风险改动已审批

## 2) Flutter 代码质量
- [ ] 组件拆分合理、状态流清晰
- [ ] 无多余 rebuild 热点
- [ ] 异常处理与空值处理完整
- [ ] 文案/交互一致性通过

## 3) 安全与数据
- [ ] 本地存储与隐私数据处理合规
- [ ] 网络错误与超时处理完备
- [ ] 数据模型变更向后兼容

## 4) 验证（必填）
- [ ] `flutter analyze` 通过
- [ ] `flutter test` 通过
- [ ] `flutter build apk --debug`（或等价）通过
- [ ] 关键页面手测通过

## 5) 交付与回滚
- [ ] PR 信息完整（目标/范围/验证/风险）
- [ ] 回滚路径明确（`git revert <commit>`）
- [ ] 结论：可提交 / 不可提交
